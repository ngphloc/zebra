Conditional Inclusion of Fragments

WOW! lets you conditionally include or hide parts of a page. In order to use conditional fragments the page needs to be in the XHTML+WOW format, not in standard XHTML. The condition is an expression using attributes of (user model) concepts and constants. The condition is included in an <if> tag. One or two <block> tags are used for the conditional fragments. They correspond to a "then" and "else" construct in programming languages. An example: 

<if expr="tutorial.usermodel.suitability==true">
<block>
This part is included when in application "tutorial"
the attribute "suitability" of concept "usermodel" has the value true.
</block>
<block>
This optional second part is included when the first part
is not included.
</block>
</if>

Conditions can contain Boolean combinations of expressions, like: 

"tutorial.pages.knowledge&gt;50 and tutorial.fragments.knowledge&lt;30"
It would have been nice to be able to write < and > instead of &lt; and &gt; but unfortunately this would confuse the XML parser.
Instead of tutorial.usermodel.suitability==true one may also simply write tutorial.usermodel.suitability because that is already a Boolean expression. In the expressions you can use the words "not", "and" and "or". Standard priority rules apply but when in doubt you can use parentheses.
If strings are used in expressions they must be put in single quotes ('), so as to not confuse the quotes with the double quotes (") aroung the whole expression.

WOW! 1.0 has an alternative way to conditionally include information: conditional objects. They allow for not only conditional but also reusable content and can be used with standard XHTML as well as with the XHTML+WOW format.

Note: Currently there is a limitation in WOW! that causes the conditional inclusion of fragments not to work within a link anchor. (So the link anchor text cannot contain a conditionally included fragment, although a conditionally included fragment can contain link anchors.) This will be fixed in the future.
