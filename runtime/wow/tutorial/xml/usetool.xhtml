Creating Concepts Using Graph Author Tool

We will see how to make simple concepts using the Graph Author Tool provided with WOW!e.

The Graph Author Tool is the main authoring tool for the concept structure. It lets you create a concept hierarchy and concept relationships of different types.
Note: The use of Concept Editor Tool and manual editing of the XML file that contains the concept structure is not recommended.

  1. First you need to start the Graph Author tool. This can be done through the Application Management Tool (AMT).
    AMT requires you to Log In as an author first. (WOW! comes with a default author called "author", with password "wow". You then see the applications of this author.
  2. Create a new application by either using the menu toolbar or using the button . We will assume that you create a new application, called example.
  3. After you created the example application you can open the application with the Graph Author. (Right click on the application for a menu of actions, or go to the Author files tab and double click on the "example.gaf" file.) After it is opened the Graph Author should look like in this screendump (but the toplevel concept will be named "example").
    Note: All the functions performed by the buttons in the toolbar can also be achieved by corresponding menu items. The functions of buttons are mentioned in their tooltips. For further reference visit the Graph Author Page of this tutoria
  4. In the left of the tool you see the concept hierarchy. Right click on the the concept named 'example' and click [Edit].
  5. What you see now is a Edit Concept Dialog Box. It contains the following fields:
    • Name : This is the name of the concept. Every concept must have a unique name, which in WOW! must be a single word (alphanumeric and starting with a letter). It is advisable to use descriptive names. The first concept should have the same name as that of the application. Let us name it 'example'
    • Description : This corresponds to the self defined short description of the concept. This is basically used for further usage and cross usability.
    • Resource : This corresponds to the page written in HTML or XHTML which this page corresponds to. It follows a strict format
      file:/name_of_the_application/relative_path_of_the_file
      Let us keep this field blank as for now.
    • Template : This drop down menu lets you select predefined concept templates included in WOW!. We select the 'abstract concept' template.
      Note: The abstract concepts don't have any resources.
    After the completion of above steps the Edit Concept Dialog Box looks like this screendump.
    Now press OK. Notice how the name of the concept has changed.
  6. Now add a concept named 'welcome' (child) by selecting 'example' (parent) and then clicking [Add New Concept] or right clicking 'example' and then selecting [Add Concept].
  7. Now fill in the following fields in the Add Concept Dialog Box
    • Name : welcome
    • Description : Welcome Page
    • Resource : file:/example/welcome.xhtml
    • Template : page concept
    This is how it looks like finally : screendump.
  8. Repeat Step 7 and add concepts 'history','basic','advanced' and 'final' respectively. Make sure to use the corresponding names in the Resource field. It is advisable to keep the name of the concept and name of the resource file as same. This screendump shows the what the hierarchy looks like.
  9. Note how the image next to 'example' in the concept hierarchy has changed. Clicking on it expands the concept, indiacating that 'welcome', 'history', 'basic', 'advanced' and 'final' are indeed its children.
  10. Now right click on 'basic' and 'advanced' concepts and select [edit]. Change the template of both of them to be [container page concept]. (We do this because we create subconcepts for them, so they are a page but there are other pages that deal with parts of the concept.)
  11. Now add concept named 'document' to the 'basic' concept by selecting 'basic' and then clicking [Add Concept] or right clicking 'basic' and then selecting
    [Add Concept]. Fill the fields as shown:
    • Name : document
    • Description : Document Tags
    • Resource : file:/example/docuement.xhtml
    • Template : page concept
  12. Repeat Step 10 and add concepts 'lists' and 'textstructure' respectively.
  13. Repeat Step 9 and Step 10 and add 2 new concepts to the 'advanced' concept. Name these concepts as 'special' and 'anchors'.
