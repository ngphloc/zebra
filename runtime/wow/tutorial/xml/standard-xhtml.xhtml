Using WOW! with standard XHTML

Pages in standard XHTML start with:
<!DOCTYPE html SYSTEM "/wow/WOWstandard/xhtml1-strict.dtd">

When using WOW! with standard XHTML you must create pages that contain a <head> and a <body part. (In the XHTML+WOW format you should not have a <head> and a <body part as they are generated automatically by WOW!.)

In the <head> part you can include a reference to an external style sheet, like <link rel="stylesheet" type="text/css" "tutorial.css" />. In the style sheet you can for instance define the presentation style of text fragments, depending on the visibility attribute of a concept.

Another possible application of using standard XHTML is that extensions other than the WOW! extension can be used. You can for instance provide the following HTML tag:
<html xmlns="http://www.w3.org/1999/xhtml" xmlns:t="urn:schemas-microsoft-com:time">
and then in the <head> add the import command
<?IMPORT namespace="t" implementation="#default#time2"?>
This then allows you to use the HTML+TIME extensions defined by Microsoft, and closely related to the timing part of the W3C SMIL standard. Using these timing extensions it becomes possible to create (adaptive) multimedia applications in which video fragments can be played either upon mouse clicks or triggered by a timer or an offset into other timed objects. It is beyond the scope of this tutorial to explain the use of HTML+TIME. Please note that when you use such extensions your application will no longer be browser-independent. (HTML+TIME currently works with Internet Explorer and with Realplayer.)
