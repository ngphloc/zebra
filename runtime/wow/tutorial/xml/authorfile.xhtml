The Graph Author ".author" Files

The ".author" files are used by the graph author to decide how to draw concept relationships of a given type. We explain the templates using the examples of knowledge_propagation and interest.

The file "knowledge_propagation.author" is as follows: 

<?xml version="1.0"?>
<!DOCTYPE author_relation_type SYSTEM 'author_relation_type.dtd'>
<author_relation_type>
    <name>knowledge_propagation</name>
    <color>none</color>
    <style>none</style>
    <properties acyclic="false" concept_hierarchy="true" />
</author_relation_type>
The file "interest.author" is: 
<?xml version="1.0"?>
<!DOCTYPE author_relation_type SYSTEM 'author_relation_type.dtd'>
<author_relation_type>
    <name>interest</name>
    <color>blue</color>
    <style>full</style>
    <properties acyclic="false" />
</author_relation_type>

The meaning of the different tags is as follows:
  • name: This is the name of the concept relationship type, as it will be shown by the graph author. To avoid confusion you should keep this name the same as the filename (without suffix).
  • color: This is the color used to draw arrows for relationships of this type. For knowledge_propagation the color is "none" because knowledge_propagation is not drawn. Other relationship types do have a color which can be black, blue, cyan, gray, green, magenta, red, or yellow.
  • style: This is the drawing style for the arrows. The style is "none" for knowledge_propagation because knowledge_propagation is not drawn. Possible styles are full, dashed and dotted.
  • properties: indicates properties of the relationships or the graph of relationships of this type.
    • acyclic: Can be true or false. (But in the case of false it is omitted.) This means that the graph formed by relationships of this type must be acyclic (or not). Cyclic relationships can potentially lead to infinite loops (but so can combinations of relationships of different types that each are acyclic but in combination become cyclic.)
    • concept_hierarchy: Can be true or false. (Again, in case of false the property can be omitted.) This means that this type of relationship coincides with the concept hierarchy. This is true for knowledge_propagation. The graph author shows the concept hierarchy in the left frame, which looks somewhat like a windows explorer tree. Relationships of other types are shown in the right frame using a graph.
    • unary: Can be true or false. (Again, in case of false the property can be omitted.) A unary relationship involves only one concept. Other relationships are binary and have a "source" and "destination", or alternatively a "parent" and "child".
