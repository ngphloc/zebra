The Graph Author Concept Template Files

The concept template files are used by the graph author for creating different concept structures (with different attributes) for different types of concepts. We explain the templates using the example of the page template.

The file "page_template.ct" is as follows: 

<?xml version="1.0"?>
<!DOCTYPE template SYSTEM 'template.dtd'>
<template>
<name>page concept</name>
<attributes>
    <attribute>
        <name>access</name>
        <description>triggered by page access</description>
        <default>false</default>
         <type>bool</type>
         <isPersistent>false</isPersistent>
         <isSystem>true</isSystem>
         <isChangeable>false</isChangeable>
    </attribute>
    <attribute>
        <name>knowledge</name>
        <description>knowledge about this concept</description>
        <default>0</default>        
        <type>int</type>
        <isPersistent>true</isPersistent>
        <isSystem>false</isSystem>
        <isChangeable>true</isChangeable>
    </attribute>    
    <attribute>
        <name>visited</name>
        <description>has this page been visited?</description>
        <default>0</default>        
        <type>int</type>
        <isPersistent>true</isPersistent>
        <isSystem>true</isSystem>
        <isChangeable>false</isChangeable>
    </attribute>    
    <attribute>
        <name>suitability</name>
        <description>the suitability of this page</description>
        <default>true</default>
        <type>bool</type>
        <isPersistent>false</isPersistent>
        <isSystem>false</isSystem>
        <isChangeable>false</isChangeable>
    </attribute>    
    <attribute>
        <name>interest</name>
        <description>is the user interested in this concept</description>
        <default>0</default>
        <type>int</type>
        <isPersistent>true</isPersistent>
        <isSystem>false</isSystem>
        <isChangeable>true</isChangeable>
    </attribute>        
</attributes>
<hasresource>true</hasresource>
<concepttype>page</concepttype>
<conceptrelations>
    <conceptrelation>
        <name>knowledge_update</name>
        <label>35</label>
    </conceptrelation>
</conceptrelations>
</template>
The meaning of the different tags is as follows:

  • The concept template starts by defining a name for the template. This is used to create the names in the list of available templates, in the dialog box for creating a new concept or for editing an existing one.
  • For each attribute there is a name, description, default value, data type (bool, int or string), and three flags to indicate whether the attribute is persistent, whether it is a system-defined attribute and whether it is changeable, meaning that it can be used in forms.
  • Concept of some types must be associated with a resource, whereas other types cannot be associated with a resource.
  • Concepts also have a concepttype which is used in the definition of the layout or look and feel of the application. Concepts created using different templates may share the same layout.
  • Concepts may have a concept relationship with themselves. The page concept has a knowledge_update relationship with itself, with a label or parameter of 35. What this particular relationship means is that the knowledge of the page will be increased up to 35 (that's the parameter) if the page is visited when it's not "suitable", and will be increased to 100 if it is visited and "suitable".
