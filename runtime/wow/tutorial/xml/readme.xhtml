What you need to know first:

Welcome to the WOW! tutorial. In order to enjoy learning about and using WOW!, please go over these instructions on how to deal with differences between computing environments and browsers, so that you can be certain you have the proper environment.

"Listen very carefully: I will say this only once!
(Quote from the TV show "Allo Allo".)

This tutorial is offered on-line, on the WOW! website but is also included in the WOW! distribution. When you installed the WOW! distribution on your own system, going through the tutorial is a good way to check whether the system is functioning properly. The tutorial is pre-installed in the distribution. This tutorial will explain how to uninstall it should you decide you no longer want it on your own server.

There are a number of things you should pay attention to when installing WOW! and using it, either as an end-user or as an author:

  • WOW! is implemented entirely in Java, and works with the Java-based webserver Tomcat and with Java servlets. You need recent versions of Tomcat and of the Java SDK to make WOW! work. On the browser side you should use recent versions (of for instance Mozilla Firefox or Microsoft Internet Explorer) to ensure full support of (X)HTML and HTTP. If you experience problems with WOW! in combination with your browser, please make sure you access the WOW! server right after starting the browser (without visiting any other pages). Some browsers, especially Internet Explorer, sometimes do not work correctly with WOW! after their internal memory has become corrupted through visits to other Websites. There should be no problem when WOW! is the first site you visit.
  • WOW! provides content adaptation by conditionally selecting pages, fragments or objects and through link adaptation by conditionally changing the color of link anchors and possibly adding icons. The adaptation is performed each time a page is requested from the server. WOW! tries to "convince" the browser to request a page from the server even when you navigate through the browsing history (using the back and forward buttons), but this is not guaranteed to work with all versions of all browsers. (At the time of writing this it does work with Firefox and Internet Explorer.) When a page is opened in a new browser window only the newly opened window shows adaptation. The other window will continue to show the previous page in its "old" state. Thus, as an author you should try to avoid situations where you really depend on adaptation when the end-user is navigating through the history or is opening new windows.
  • The link adaptation is performed through the use of a style sheet. The browser must be configured to accept colors which the html page requests. With most browsers this is the default setting. In this tutorial you will see that links are sometimes blue and sometimes purple. These colors are determined by the style sheet, not by the browser's history mechanism. Links may also be black (actually dark gray) meaning that according to the adaptive application the user is not ready to follow them. You may also see bright red or dark red links; they point to pages that are not part of the tutorial.
  • WOW! pages can (but need not) have a header and/or footer, showing a progress report and access to forms for changing the user profile, color preferences or password, or to log out. If this header does not show up when you expect it, it probably means that the page was not served through WOW! but straight through the web server. (As a result there is then no adaptation.)
  • WOW! runs on any full and recent implementation of the Java 2 platform. It is frequently tested on Microsoft Windows (currently XP) and Linux (mostly Fedora). Linux is the prefered platform, not just because it is Open Source like WOW! itself, but also because it is better at multitasking. Any "plain" PC (or even a laptop computer) will be adequate for a small WOW! server, resulting in sub-second response times.
