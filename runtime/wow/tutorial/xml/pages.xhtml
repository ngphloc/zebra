The WOW! Page Format

Pages in WOW! applications are written in XHTML, with or without an extension for special WOW! tags (explained below). Every WOW! page that uses the special tags must start with the following line: 

<!DOCTYPE html SYSTEM "/wow/WOWstandard/xhtml-wowext-1.dtd">
(where the /wow prefix is the path parameter in the WOW! configuration). The content of the page uses a default namespace of xhtml: 
<html xmlns="http://www.w3.org/1999/xhtml">
...
</html>

The content of a page should follow the XHTML or the XHTML+WOW format. (WOW! does not validate the syntax, but it does use an XML parser and will therefore not accept pages that are not valid XML.) We have a short description of the differences between HTML and XHTML and also a description on how to use WOW! with standard XHTML. WOW! pages (in the XHTML+WOW format) start with the <html> tag but do not have a <head> or <body> tag, because these are automatically generated. The header contains an automatically generated (link to a) style sheet that defines the link colors, and the body tag defines the background for the page. WOW! pages may contain four types of special elements: tags to generate certain information, typically used in headers and footers, tags to conditionaly include fragments, conditional objects and conditional links.

  • Headers and footers are examples of the unconditional inclusion of objects, through the object tag, with a data attribute that is a URL pointing to the file to be included, for example: 
    <object data="../header.xhtml" type="text/wow" />
    There are special tags that are typically used in headers and footers but that can in fact be used anywhere in WOW! pages. Headers and footers are optional.
  • Fragments are conditionally included using the <if> tag. The content of a fragment must be a syntactically correct piece of XHTML+WOW code (possibly including other fragments), in which all the tags are properly nested.
  • Content can also be conditionally included using conditional objects. These are defined through the standard XHTML <object> tag, with the special type of text/wow and a name attribute that is a URL that refers to a concept. The advantages over conditional fragments are that no extension of XHTML is needed and that the same piece of content can be included in different pages without having to copy it.
  • WOW! distinguishes three types of links: conditional, unconditional and external.
    • Conditional links are of the class "conditional". The destination can be a concept or a page. (The application's concept structure connects concepts to pages. In most cases it makes no difference whether you create a link to a concept or to the corresponding page.) 
      <a href="tutorial.conceptname" class="conditional">...</a>
      or 
      <a href="conceptname" class="conditional">...</a>
      (in which the current application is assumed), or 
      <a href="pagename.xhtml" class="conditional">...</a>
      or 
      <a href="http://site/pagename.xhtml" class="conditional">...</a>
      are conditionally shown to or hidden from the end-user. The condition is part of the application's concept structure. In the typical (but not required) color or link presentation scheme the link color of "desired" links depends on the "visited" status of the link destination. The default color scheme is blue (desirable, unvisited), purple (desirable, visited) and black (undesirable). WOW! allows different and much more complex color schemes as well. The end-user can choose different colors. Whatever the color scheme is, the "ColorConfig" form gives the end-user access to all the colors (but not to the conditions for choosing them).
      It is possible to associate more than one page with a concept. This is done in exactly the same way as with conditional objects. Depending on conditions a different page can be shown for a single concept. We call this adaptive link destinations. When a new user registers for this tutorial the first ("welcome") concept has an adaptive link destination. The initial "readme" page you see as a new user is shown only once. As of the second visit a "welcome" page is shown instead.
    • Links can be declared as "unconditional", in which case their presentation only depends on the "visited" status. These links can point to a concept or a page, just like conditional links.
    • Links that are not conditional or unconditional are considered to be "external" links. They can be recognized by a different color (and possible underlining). By default the colors are bright red (unvisited) and dark red (visited).
