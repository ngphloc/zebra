XHTML versus HTML

WOW! 1.0 uses pages written in XHTML. Pages are parsed using an XML parser. In many cases when an end-user (hopefully you as an author trying out your application) tries to load a page with an XML syntax error the browser will display an error message (from the XML parser) indicating what is wrong with the page. Unfortunately there is nothing the end-user can do to rectify the problem. The author of the page has to correct the error, so as an author you should always check the pages for XML errors.

For experienced HTML authors there are a few things you should know in order to write (mostly) correct XHTML pages that will work in XHTML browsers as well as older HTML browsers:

  • All tags have to be properly nested. In HTML there is automatic ending of tags, like in the following construct: 
    <ul>
<li> ... <li> ... <li> ... 
</ul>
    Where each list item is ended automatically when the next list item starts or when the whole list ends. in XHTML you have to explicitly end all tags: 
    <ul>
<li> ... </li> <li> ... </li> <li> ... </li>
</ul>
  • In HTML there are a number of tags that have no end tag, like 
    <br>
    In XHTML all tags must be ended. There are two ways to do this: 
    <br></br> or <br />
  • Older browsers ignore explicit end tags like </br> but may get confused by the abbreviated form <br/>. A solution is to add a space between the tag name and the /, as in <br />.

Unless you are creating an application for a known audience with a single known browser it is best to try all pages on several browsers, including some new ones (like Mozilla Firefox) and some old ones (like Netscape version 4).

Note: WOW! can actually handle HTML pages, by using a converter from HTML to XHTML (called JTidy). This conversion may not be perfect, but it works for most HTML pages.
